# Operations runbook

Quick reference for running the Ubuntu reposerver. All commands run on the
server host as the `aptly` user unless noted (`sudo -u aptly ...`).

## Mental model

- **Mirror** — a local copy of an upstream pocket (e.g. `noble-security`).
  Mutable; `sync` updates it.
- **Snapshot** — an immutable, dated freeze of a mirror
  (`noble-security-20260620-020000`). This is what you publish.
- **Stage** — a published channel (`dev`/`test`/`prod`) that points at one
  snapshot per pocket. Clients subscribe to a stage. Promotion = re-pointing
  a stage at a newer snapshot.

State pointers live in `/srv/aptly/data/state/<stage>/<suite>` and every
change is appended to `/srv/aptly/data/state/history.log`.

## Routine: nightly (automatic)

`aptly-sync.timer` → `aptly-sync.service` → `scripts/daily-sync.sh`:
sync mirrors → snapshot → publish to `dev` → cleanup. Inspect it:

```bash
systemctl status aptly-sync.timer
systemctl list-timers aptly-sync.timer
journalctl -u aptly-sync.service -n 100 --no-pager
```

## Routine: promote validated patches

1. After a nightly run, `dev` has the freshest snapshot. Point a canary box
   at `dev` and verify it upgrades cleanly.
2. Promote to `test`, soak across your test fleet:
   ```bash
   sudo -u aptly repoctl promote dev test
   ```
3. Promote to `prod`:
   ```bash
   sudo -u aptly repoctl promote test prod
   ```

Promote a single pocket (e.g. ship only security fixes early):

```bash
sudo -u aptly repoctl promote test prod noble-security
```

## Rollback

Promotion just re-points a stage at a snapshot, and old snapshots are kept
(`SNAPSHOT_KEEP`, default 6). To roll `prod` back:

```bash
# List recent snapshots for the affected pocket, newest first:
sudo -u aptly aptly snapshot list -raw | grep '^noble-updates-' | sort -r | head

# Re-publish the previous good one to prod (re-points the stage):
sudo -u aptly bash -c '
  source /opt/ubuntu-reposerver/scripts/lib.sh
  publish_or_switch prod noble-updates noble-updates-20260613-020000
'
```

Clients pick it up on their next `apt-get update`.

## Disk management

```bash
du -sh /srv/aptly/data/*            # where space goes
sudo -u aptly repoctl cleanup       # drop old snapshots + gc the pool
```

If disk is tight: lower `SNAPSHOT_KEEP`, trim `COMPONENTS` (e.g. drop
`multiverse`), keep `INCLUDE_BACKPORTS=false`, or mirror security-only by
removing the `-updates` line from `suites()` in `scripts/lib.sh`.

## Add a release or architecture

1. Edit `config/reposerver.env` (`RELEASES`, `ARCHES`, etc.).
2. Recreate mirrors and sync:
   ```bash
   sudo -u aptly repoctl init
   sudo -u aptly repoctl sync
   sudo -u aptly repoctl snapshot       # publishes new pockets to dev
   sudo -u aptly repoctl promote dev test
   sudo -u aptly repoctl promote test prod
   ```

> Changing `ARCHES` only affects mirrors created afterwards. To add an
> architecture to an existing mirror, drop and recreate it:
> `aptly mirror drop <suite>` then `repoctl init`.

## Use a closer/faster upstream

Set `ARCHIVE_URL` to a regional mirror (e.g.
`http://us.archive.ubuntu.com/ubuntu`) in `config/reposerver.env`, then
`repoctl init` (new mirrors) — existing mirrors keep their original URL
until recreated. Keep `SECURITY_URL` on `security.ubuntu.com` for timely
security updates.

## Signing key

- Generated by `install.sh`; id stored in `/srv/aptly/gpg-key-id`.
- Public key exported to the web root: `http://<host>/reposerver.gpg` (and
  `.asc`). Clients install it via `client/setup-client.sh`.
- **Back up** `/srv/aptly/.gnupg` and `/srv/aptly/gpg-key-id`. If the private
  key is lost, every client must re-trust a new key.

## Troubleshooting

| Symptom | Check |
|---------|-------|
| Client: `NO_PUBKEY` / signature error | Key not installed: re-run `setup-client.sh`, confirm `Signed-By` path |
| Client: 404 on `update` | Stage/suite not published yet — `repoctl status`; promote into that stage |
| Sync fails | Upstream reachable? `curl -I $ARCHIVE_URL/dists/noble/Release` |
| `aptly publish switch` errors | Snapshot exists? `aptly snapshot list -raw` |
| Nginx not serving | `nginx -t`, `systemctl status nginx`, root = `/srv/aptly/data/public` |
